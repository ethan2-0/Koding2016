How does Spaghetti Compute Work


The Spaghetti Compute system works by taking problem requests from customers,
and then distributing portions of the problem over a network of machines,
which we call "botnet". These machines then take these subproblems, find their
solutions by using computation power from a browser, and then sends it back to Spaghetti
Compute’s servers, who then return the results to the customer. This
technique of dividing up a large problem into smaller ones is known as
mapreduce, and is a commonly used dynamic programming procedure.

Now that you understand what Spaghetti Compute is doing in a general form,
let’s explain how we get distributed computation to work. First, we need to
understand how Spaghetti Compute structures problems. When customers submit
problems to be solved, they use our client Application Protocol Interface (API)
to splice the problem into smaller subproblems. These subproblems are called
work units, and are given to different machines across the botnet.

Each machine in the Spaghetti Compute system is actually a browser
visiting a webpage with a Spaghetti Compute JavaScript (JS) file embedded
onto it. This JS file communicates with Spaghetti Compute servers via
HyperText-Transfer-Protocol (HTTP) requests. Machines in the botnet then
ask Spaghetti Compute’s servers for a work unit to solve. Once this
request has been validated, the machine is assigned a work unit to solve.
When the problem is solved, it sends the solution back to the servers at
Spaghetti Compute.

Now, the computer on which the browser is running can belong to just
about anyone in the world. Anybody who is visiting a website that has a
Spaghetti ComputeJS file is a part of the botnet. This is the real beauty
of the Spaghetti Compute system. Instead of seeing advertisements, surveys,
or subscription requests on websites, people will only get a popup telling
them that their computer is being used to solve a work unit. Since our
services run on web workers, the calculations are performed off the main
thread - this means that clients do not even receive delay when their
computers are being used.
